# 实验项目配置化方案

为了实现实验项目与子模块（组件）关联关系的动态配置，我将执行以下步骤：

## 1. 数据库变更
创建 `PerformanceItemConfigs` 表，用于存储实验项目的配置信息：
- `ID`: 主键
- `ItemName`: 项目名称（如"初粘性/持粘力/高低温"）
- `ItemCode`: 项目代码（唯一标识，如 "Adhesion"）
- `ComponentType`: 对应的子模块组件类型（如 "AdhesionForm"）
- `SortOrder`: 排序字段
- `IsEnabled`: 是否启用

并预置现有的 4 种实验项目数据。

## 2. 后端 API 开发
在 `server/routes/performanceInspection.js` 中添加以下接口：
- `GET /configs`: 获取所有启用的配置列表（用于下拉菜单）。
- `GET /configs/all`: 获取所有配置（用于管理）。
- `POST /configs`: 新增配置（管理员）。
- `PUT /configs/:id`: 修改配置（管理员）。
- `DELETE /configs/:id`: 删除/禁用配置（管理员）。

## 3. 前端改造
### 3.1 `TestItemsManager.vue`
- 移除硬编码的下拉菜单项。
- 在 `onMounted` 中调用 API 获取配置列表。
- 根据配置动态渲染下拉菜单。
- 建立 `ComponentType` 字符串到实际 Vue 组件对象的映射机制。
- 添加"配置管理"入口（仅管理员可见），点击打开配置管理弹窗。

### 3.2 新增 `PerformanceConfigDialog.vue`
- 实现配置管理界面，允许管理员：
    - 查看现有配置列表。
    - 新增实验项目：输入名称，选择绑定的子模块（从预定义的组件列表中选择）。
    - 编辑/删除/排序现有项目。

## 4. 预期效果
- 管理员可以自由添加新的实验项目名称，并指定其使用哪个录入模板（子模块）。
- 例如，可以新增一个 "高温持粘性" 项目，并复用 "AdhesionForm" 模板。
- 普通用户在添加实验时，将看到管理员配置好的最新列表。
